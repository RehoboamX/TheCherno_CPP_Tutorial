# Lesson9 函数
#### 函数就是我们设计的专门用来执行特定任务的代码块。在后面我们学习类(class)的时候，那里面的代码块称为方法，要注意区分。当我们提到“函数”的时候，我们明确的表示这是不属于c++类里面的东西。使用函数的主要作用在于防止代码的重复。
(1)函数的定义由**返回类型**，**函数名**，**参数列表**和**函数体**四部分组成。函数列表可以为空，这样函数不传入任何参数。函数也可以不返回任何东西，只要把返回类型定义成**void**即可。  

(2)每次我们调用函数时（*）。编译器会生成一个**call**指令。这意味着在运行的程序中，为了调用一个函数，计算机需要创建一个**堆栈**结构，像是参数这样会被推进堆栈。我们还需要将一个叫做**返回地址**(**return address**)的东西推进堆栈以保证**IP**最后能找到调用函数的位置。当执行调用函数的指令时，计算机中的**指令指针寄存器**(**IP**)会寻址到二进制文件的定义函数的位置。最后为了将压入堆栈的return结果返回，**IP**会寻址回最初调用函数的位置。执行函数的过程实际上会在内存中来回跳跃，需要额外的时间。因此，不要将任何用到的功能都一律的封装成函数，这会减慢我们程序的运行。关于什么情况下需要定义函数需要一些经验，但基本上都是需要多次做一项任务时为防止代码重复需要定义函数。    
　　注：上面星号表示上面的陈述都是建立在编译器将我们定义的函数看作实际的函数而不做内联inline工作的假设下成立的，对于inline我们在后面的课程中会有介绍。  

(3)在**main**()函数中，返回类型只能是**int**，但是函数中我们不人为return任何值程序也不会出错。所以我们定义其他返回类型int的函数，也可以不返回任何值吗？答案是否定的。有返回类型的函数必须返回相应类型的值。**main**()函数的特例是因为其是特殊的，如果不返回任何值，程序会自动返回0。但函数必须返回值是在**debug模式**下成立的，如果我们在***解决方案配置***中将debug模式改为**release模式**，那么普通函数不返回任何值也能编译通过了。这里并不是说不返回任何值的行为是正确的，当我们需要返回值时其一样会报错。只是**release模式**下编译器会忽视这种错误，但这种错误我们仍需要避免。

(4)我们经常将函数分解为**定义**(**defination**)和**声明**(**declaration**)两个部分。**声明**通常被写在在**头文件**中，而**定义**通常被写在**翻译单元**或是**cpp**文件中。
